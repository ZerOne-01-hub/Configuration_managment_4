Практическая работа №4

Дисциплина: Конфигурационное управление

Вариант 5

1. Цель работы

Целью практической работы является разработка инструмента командной строки для обработки учебного конфигурационного языка.
Инструмент выполняет синтаксический разбор входного текста, вычисляет константы на этапе трансляции и преобразует данные в формат JSON.

2. Постановка задачи

Необходимо реализовать программу, которая:

принимает входной текст на учебном конфигурационном языке из файла, путь к которому задаётся ключом командной строки;

выполняет лексический и синтаксический анализ входного текста;

выявляет синтаксические и семантические ошибки с выдачей сообщений;

поддерживает объявление и вычисление констант;

выводит результат работы в формате JSON в стандартный поток вывода;

использует специализированный инструмент синтаксического разбора.

3. Синтаксис учебного конфигурационного языка
Комментарии

Многострочные комментарии имеют вид:

/*
текст комментария
*/

Числа

Числовые значения задаются в формате:

\d*.\d+

Строки

Строковые значения имеют вид:

@"Это строка"

Имена

Имена идентификаторов задаются регулярным выражением:

[_a-zA-Z]+

Массивы

Массивы задаются в круглых скобках:

( значение, значение, значение, ... )

Допускаются пустые и вложенные массивы.

Константы

Объявление константы:

имя is значение

Вычисление значения константы на этапе трансляции:

?[имя]

Результатом вычисления является значение соответствующей константы.

4. Используемые технологии

В ходе выполнения работы использовались следующие технологии и инструменты:

Python 3 — язык реализации программы;

PLY (Python Lex-Yacc) — специализированный инструмент для лексического и синтаксического анализа;

стандартная библиотека Python json — для формирования выходных данных в формате JSON;

Git и GitHub — для контроля версий и хранения исходного кода.

5. Архитектура программы

Программа состоит из следующих модулей:

lexer_conf.py

Модуль лексического анализа.
Содержит описание токенов языка, регулярные выражения для чисел, строк и идентификаторов, обработку комментариев и лексических ошибок.

parser_conf.py

Модуль синтаксического анализа и семантической обработки.
Содержит грамматику учебного конфигурационного языка, хранение объявленных констант, вычисление значений конструкций ?[имя], а также обработку синтаксических и семантических ошибок.

main.py

Модуль интерфейса командной строки.
Осуществляет разбор аргументов, чтение входного файла, запуск синтаксического анализа и вывод результата в формате JSON.

6. Примеры использования

Запуск программы осуществляется из командной строки:

python main.py -i <путь_к_файлу>

Пример запуска:

python main.py -i tests/basic.conf

7. Тестирование

Для проверки корректности работы программы подготовлены следующие тестовые файлы:

basic.conf — тест основных конструкций языка;

domain_service.conf — пример конфигурации из предметной области сервисов;

domain_game.conf — пример конфигурации из предметной области игровых настроек.

Каждый тест запускается вручную через командную строку.

8. Обработка ошибок

Программа выявляет и обрабатывает следующие типы ошибок:

лексические ошибки;

синтаксические ошибки;

семантические ошибки (повторное объявление констант, использование несуществующих констант).

При возникновении ошибки выводится диагностическое сообщение, после чего программа завершает выполнение с ненулевым кодом возврата.

9. Вывод

В результате выполнения практической работы разработан инструмент командной строки, полностью соответствующий требованиям варианта 5.
Программа корректно выполняет синтаксический разбор учебного конфигурационного языка, вычисляет константы на этапе трансляции и преобразует данные в формат JSON с использованием специализированного инструмента синтаксического анализа.
